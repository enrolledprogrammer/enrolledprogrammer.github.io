交易| 自制指标｜ 高胜率交易模型:下单前的最后一步-盈亏比

发表于 2021-05-19 更新于 2021-07-14 分类于 交易 ， 自制指标 ， 高胜率

前言：

我总结了一下我的失败案例。不出意外，每次都是因为固执的做反了趋势，并且忍受了持续性亏损。到了2020年，我的亏损单主要来源于期权操作。这是一个好的信号，也是一个幸运的指标。

伴随着2020年对基本面的分析，股灾的重筹投注以及压住全球放水让我们有了不赔的优势。但是对动能和盈亏比的不了解。让我没有能够赚到足够的资本。所以我写了这一篇，主要是抛砖引玉，寻找一种最简单盈亏比的策略。

写在前面的话：

为什么一定是盈亏比？

好的入场点能让在出场点上有更好的心里优势。买在高点的确是策略之一，但是后面每一个单位时间的价格形态（K棒）都会造成心理压力。如果不笃信则会卖在起涨点，如果笃信则会造成大额亏损。

如果是顺大周期逆小周期会怎么样呢？

如果逆势的小周期变成了大周期，买的起涨点却变成了卖压点。对于新手朋友来说，会怀疑这个理论。但是这个理论却是让不知信息点散户获胜的唯一法门。（从赔率角度出发）

在玩德州扑克此类投资游戏的时候，在7到9人局面对CUTOFF位置玩家下注，和SB位置玩家下注时候，BB位（视角）的玩家往往会跟注。如果你在听直播的话，会常常听到解说说到：按照赔率来说，是合适的。

面对德州扑克这种负和游戏（庄抽水），股票市场也是一个类似游戏（券商抽水）。不论大家是资金管理的信徒（海龟交易法），还是技术指标派（《走进我的交易室》），或者是分析基本且等待股灾的投资者。在多个市场中，我们寻找可以投资的标的。目的是，获得高赔率的交易。

请注意，是高赔率而不是高概率获胜。

倘若在1987年选择一个非时间耗尽的投资标的，拥有者最小交易规模，则你拥有了100%的概率。但是这样的投资，与购买黄金，国债，房产，甚至叛逃他国赚取的美金多寡，只有你自己最清楚。

正文：

我怎么定义盈亏比？

我把盈亏比定义的十分简单，甚至简单的可笑。因为这是我下单前的最后一步。下单前我应该考虑好突破区间，资金流向，基本面向好，多头空头能量等。最后一步的盈亏比起到了过滤投资点位的目的。另外，我需要在很小的时间刻度上进行过滤。

所以，我期望这个指标可以有如此大作用：

盈亏比=（高点-开盘价格）/ （低点-收盘价格）

考虑到情绪因素，一个悲观的投资者认为资金不会卖在高点，那么公式应该改为：

盈亏比=（高点-开盘价格 -波动）/ （低点-收盘价格+波动）

但是我的指标是为了过滤购买机会而不是让我错过机会，所以我宁愿相信我有能力卖在高点，而不是放弃一些仅仅是波动率的影响就轻易放弃的机会。

盈亏比=（高点-开盘价格 +波动）/ （低点-收盘价格-波动）

TR1:=MAX(MAX((HIGH-LOW),ABS(REF(CLOSE,1)-HIGH)),ABS(REF(CLOSE,1)-LOW));

X1:=IFF(CLOSE =LOWEST + K* ATR1, HIGHEST-CLOSE + K* ATR1,ABS((HIGHEST-OPEN + K* ATR1)/(CLOSE-LOWEST - K* ATR1)) )

如果你对公示敏感的话，会想问我高点地点如何计算：

高点= MAX（N日最高点，N日EMA）

低点= MAX（N日最低点， N日EMA）

由于依靠EMA交易的人数太多，所以不得不考虑他们在到达价格时候的买入和卖出。特别是，这里的他们并不是人的情况下（量化的机器）。

所以我可以用大周期做一个HIGHEST，也可以用小周期做一个HIGHER。然后，得到了一个大周期的盈亏比，和小周期的盈亏比。

然后，如果我们遇到了盈亏比都很合适的点，我简单的认为LN（X1）>1时候是最好的观察点，当这个值大于1的时候买入，当这个值小于1的时候卖出。毕竟自然对数被设计出来就是为了金融投资的。

      • 创建createWorkInProgress()

        • 通过复用current.alternate
        • why？根据setState规则
          • 对于第一次setState 保存的就是上一次的current树。这时候虽然有DOM树，但是每一个FiberNode还没有与状态 current.alternate 互相绑定。所以第一次setState时，会创建另一个Fiber树 （T1）。
          • 第二次setState。对于这一次来说，在之前创建的T1树 current.alternate 绑定到之前到current（T0）树。Current指的是T1时刻建立的Fiber树，current.alternate 是T0时刻建立的Fiber树。T1时刻的树也就是当前页面正在渲染的结果。此次更新是在T0时刻的Fiber树上进行更新。
          • 垃圾回收：因为链表存在，alternative指向的T0时刻的树并没有被清空。
        function createWorkInProgress(current, pendingProps){
    //复用current.alternate
    let workInProgress = current.alternate
    // workInProgress 初始化
    if(!workInProgress){
        workInProgress = createFiber(current.tag, pendingProps, current.key)
        workInProgress.type = current.type
        workInProgress.stateNode = current.stateNode
        // 要让workInProgress <-> current
        workInProgress.alternate = current
        current.alternate = workInProgress
    }else{
        //
        workInProgress.pendingProps = pendingProps
        workInProgress.effectTag = NoWork
        workInProgress.fisrtEffect = null
        workInProgress.lastEffect = null
        workInProgress.nextEffect = null
    }
    // 保证updateQueue同步
    workInProgress.updateQueue = current.updateQueue

    // 保存旧的状态
    workInProgress.child = current.child
    workInProgress.memorizedState = current.memorizedState
    workInProgress.mermorizedProps = current.memorizedState
    workinProgress.sibling = current.sibling
    workinProgress.index = current.index
    return workinProgress
}

继承有多少种方法

来源
自己的认识：ES5里面，对于把 function 看成了构造函数，由于原型设计模式的原因。新的Obj被看成为一个原型Obj的拷贝，具体实现方法是Ojbect.create() 方法。
我们把function 定义的类分为了实例部分（function内部定义的方法或属性），和原型部分（通过prototype附加的属性或方法）。
如果通过function 新建一个Object：

  1. 原型设计模式会依据模版Clone一个新对象。新对象即实例，有自己属性方法。
    • 每一个实例的属性方法私有。也就是说，通过Object.create()方法，function内部定义的方法回在stack上生成（有自己的地址空间）。
  2. 原型
    • 通过prototype赋予的属性会共享存储空间。也就是说，默认使用指针方式将新对象的原型方法指向了prototype（这个指针是_proto）

Tips:
生成实例对象的常用方法，就是使用new命令，让构造函数返回一个实例。但是很多时候，只能拿到一个实例对象，它可能根本不是由构建函数生成的，那么能不能从一个实例对象，生成另一个实例对象呢？

JavaScript 提供了Object.create方法，用来满足这种需求。该方法接受一个对象作为参数，然后以它为原型，返回一个实例对象。该实例完全继承继承原型对象的属性。

寄生式继承的思路与(寄生)构造函数和工厂模式类似, 即创建一个仅用于封装继承过程的函数,该函数在内部以某种方式来增强对象,最后再像真的是它做了所有工作一样返回对象

寄生继承的缺陷在于只能作为继承的最末尾的一环，如果再有人继承这个，则其添加的方法不能被继承。

new 运算符做了什么

就干了三件事。

new 做了三件事情：

  1. 创建一个空对象，并且 this 变量引用该对象，同时还继承了该函数的原型。
  2. 属性和方法被加入到 this 引用的对象中。
  3. 新创建的对象由 this 所引用，并且最后隐式的返回 this.

这里最关键的是重新绑定this 指针，指向了最后面的那个类。如果没有这个步骤，this指针指向父亲。则子类的实例可以修改父类的值。
